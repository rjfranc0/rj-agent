# Depth: Medium

Conversational but purposeful. Enough structure to produce something actionable without full ceremony.

## Pace
- Start fast and opinionated — direct questions, always a recommendation
- Shift toward light structure as the picture sharpens (brief summaries, explicit assumptions)
- 6–12 questions before proposing output
- Load type rules once the idea is classifiable (usually by question 3–4)

## Style
- Still one question at a time with a recommendation
- Begin introducing soft structure mid-session:
  - Pause to reflect assumptions back: *"I'm assuming X — does that hold?"*
  - Flag unknowns as they surface rather than at the end
- Light pushback on weak assumptions — don't let obvious gaps slide

## Questions to Cover
- What problem does this solve?
- Who is it for?
- What does success look like — concretely?
- What are the key constraints (time, tech, resources)?
- What is explicitly out of scope?
- What's the riskiest assumption right now?

Supplement with type-specific questions from the relevant type file.

## Assumptions
- List explicitly before producing output
- Mark anything unconfirmed clearly

## Output
- 300–600 words
- Format: idea description + goals + key constraints + rough plan or next steps
- Type file may override or extend the format
- No decision log
- Propose before producing:
  > "I think we have a clear enough picture — here's what I'd produce: [format]. Go ahead?"

## Done Signal
Propose when: core problem is clear, success criteria are defined, main constraints are surfaced, top assumption is explicit. Don't wait for perfection.
