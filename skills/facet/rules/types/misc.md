# Type: Misc

Catch-all for ideas that don't fit a defined type. Unconstrained but still purposeful.

## When This Applies
- The idea spans multiple types without a clear dominant one
- The domain is personal, philosophical, organizational, or otherwise uncategorized
- No specific type file exists yet

## Domain Questions
No fixed list — follow the depth rules and use judgment. Focus on:
- What's the actual output the user needs?
- Who else is affected?
- What would make this actionable?

## Preferred Output
Infer from context. When ambiguous, ask:
> "What would be most useful at the end — a summary, a plan, a document, something else?"

## Output & Handoff
No fixed handoff target. Describe the idea domain, the inferred output type, and any constraints surfaced. Keep it open — the destination shouldn't assume a domain.

## Notes
If this idea has a clear enough shape that misc can't properly serve it, flag it immediately — don't wait: *"This feels like it could use its own type. Want to define one in a separate discussion?"*
