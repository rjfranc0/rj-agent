# Design — Non-Negotiables

These apply to every output, every context, no exceptions.

## Commit to a direction

Pick one. Execute it fully. Hedging produces nothing memorable.
A refined minimal design and a bold maximalist one are both valid — the failure mode is neither.

## Design and code are inseparable in React

The composition pattern you pick constrains what's animatable.
The layout choice shapes the component API.
The motion approach affects state structure.
Decide them together — not design first, hand off, then code.

## Creativity and functionality are independent axes

High creativity does not mean low functionality. A portfolio can be
extravagant and still usable. A dashboard can be creative and still clear.
Both axes are calibrated per context — see `thinking.md`.

## Hard NOs — never produce these

- Generic AI aesthetics: predictable layouts, cookie-cutter patterns, no point of view
- Purple gradients on white backgrounds
- Inter, Roboto, Arial, Space Grotesk, or system fonts — ever
- Animations that don't serve the design intent
- Glassmorphism as decoration (only valid as a depth tool)
- A design that looks like it could have been generated for someone else

## The standard

Every output should be defensible on Awwwards — within its context.
A functional dashboard isn't less ambitious than a portfolio. It's differently ambitious.
The question is always: is this memorable, or just correct?
